# Viewcodex 项目简报

## 项目要做什么

Viewcodex 是一个轻量级的 Codex CLI 可视化工作台。

它不是 IDE，也不是聊天网页，更不是要替代 Codex CLI。它的目标是让使用 Codex CLI 的日常流程更舒服：

- 更方便地选择和切换项目。
- 清楚看到当前是否有活跃 Codex 进程。
- 能直接进入某个线程或会话的 CLI。
- 能在运行前选择 skill、模型和是否每次任务后提交 Git。
- 能为每个项目配置“启动前必须阅读”的规范和项目文档。

## 第一版核心流程

1. 打开 Viewcodex。
2. 选择一个项目目录。
3. Viewcodex 读取用户为该项目配置的规范文档和项目阅读文档。
4. 用户选择 skill。
5. 用户选择模型。
6. 用户选择是否每次任务完成后自动提交 Git。
7. 用户查看是否有活跃 Codex 进程。
8. 用户输入需求并点击确认。
9. Viewcodex 启动 Codex CLI。
10. 用户点击线程或会话，进入真实 CLI。

## 必须做

- 项目目录选择和管理。
- 已有 skill 选择。
- 模型选择。
- 简单 Git 提交开关。
- 活跃 Codex 进程可见。
- 点击进入线程或会话 CLI。
- 上下文使用量显示，能获得时显示 K tokens。
- 项目级启动阅读文档配置。
- 每次对某个项目开启新会话前，必须先读取该项目配置的规范和项目文档。

## 启动阅读功能

Viewcodex 的界面中需要有一个“项目文档 / 启动规范”配置区。

用户可以为每个项目配置：

- 全局规范文档，例如 `AGENTS.md`、`VIEWCODEX_BRIEF.md`。
- 项目说明文档，例如 `README.md`、`docs/project.md`。
- 额外上下文文档，例如需求文档、接口文档、业务规则。

启动文档必须能在界面中选择和创建。选择已有文档时，应从项目目录中挑选 Markdown 文件；创建新文档时，应在项目内生成文件并加入启动文档列表。

当用户选择项目并准备启动新 Codex 会话时，Viewcodex 必须先读取这些文档，把它们作为启动上下文或启动前检查的一部分。

如果配置的文档不存在，界面应提示用户修复路径，而不是静默忽略。

## 明确不做

第一版不做：

- 完整 IDE。
- 内置代码编辑器。
- 插件市场。
- 复杂 MCP 下载中心。
- 云同步。
- 账号系统。
- 替代 Codex CLI 的 Web 聊天界面。
- 多人协作。
- 高级项目索引。
- 复杂数据看板。

## 推荐技术栈

第一版优先使用：

- Electron
- React
- TypeScript
- Vite
- xterm.js
- node-pty
- Zustand
- Tailwind CSS
- lucide-react
- 本地 JSON 配置

## 产品原则

核心原则：

```text
快速打开 -> 选择项目 -> 选择/创建启动文档 -> 读取项目规范 -> 选择 skill / 模型 -> 查看活跃 Codex -> 运行或回到 CLI
```

不能明显改善这个流程的功能，都应该延后。
