# Viewcodex 实现规划

## 1. 目标

做一个简单、快速、日常可用的 Codex CLI 可视化工作台。

第一版只围绕一个核心体验：

```text
选择项目 -> 选择/创建启动文档 -> 读取项目文档 -> 选择 skill / 模型 -> 查看活跃 Codex -> 启动或进入 CLI
```

## 2. 第一阶段：项目骨架

目标：跑起来一个可以打开的桌面应用。

建议技术：

- Electron
- React
- TypeScript
- Vite
- Tailwind CSS
- Zustand

交付内容：

- 应用主窗口。
- 基础布局。
- 本地配置文件读写。
- 项目选择入口。
- 最近项目列表。

本地配置建议路径：

```text
~/.viewcodex/config.json
```

## 3. 第二阶段：项目与启动文档配置

目标：让用户能配置每个项目启动前必须阅读的文档。

功能：

- 添加项目目录。
- 从界面选择项目目录。
- 为项目配置规范文档列表。
- 为项目配置项目说明文档列表。
- 从项目目录中选择已有 Markdown 文档。
- 在项目目录中创建新的规范或项目说明文档。
- 检查文档是否存在。
- 在启动新会话前读取这些文档。

配置结构建议：

```json
{
  "projects": [
    {
      "name": "Viewcodex",
      "path": "/Users/benfit/code/Viewcodex",
      "startupDocs": {
        "required": [
          "VIEWCODEX_BRIEF.md",
          "AGENTS.md"
        ],
        "optional": [
          "README.md",
          "docs/viewcodex-visual-cli-requirements.md"
        ]
      }
    }
  ]
}
```

启动规则：

- 必读文档不存在时，不启动新会话。
- 可选文档不存在时，显示提醒，但允许继续。
- 文档读取结果应进入本次 Codex 启动上下文。

## 4. 第三阶段：Codex 运行入口

目标：从界面启动 Codex CLI。

功能：

- 输入需求。
- 选择项目。
- 选择 skill。
- 选择模型。
- 选择是否每次任务完成后自动提交 Git。
- 点击确认后启动 Codex CLI。

实现建议：

- 使用 `node-pty` 启动 shell / codex。
- 使用 `xterm.js` 显示终端。
- Codex 仍然保持真实 CLI 交互。

## 5. 第四阶段：活跃进程与线程入口

目标：用户能看清当前有没有 Codex 正在运行，并能进入对应 CLI。

功能：

- 显示活跃 Codex 进程。
- 显示可检测到的项目路径。
- 显示运行状态。
- 支持点击进入对应终端。

第一版先追踪 Viewcodex 自己启动的 Codex 会话。系统里外部启动的 Codex 进程可以后续再扫描接入。

tmux 第一版处理方式：

- 先检测已有 tmux session。
- 支持进入已有 session。
- 不做复杂 tmux 编排。

## 6. 第五阶段：体验打磨

目标：让工具每天用起来顺手。

功能：

- 最近项目。
- 最近会话。
- 默认模型。
- 默认 skill。
- 默认 Git 行为。
- 启动文档状态提示。
- 启动文档选择和创建入口。
- 简洁错误提示。

## 7. 第一版验收标准

第一版完成时应满足：

- 可以添加和切换项目。
- 可以为项目配置启动前阅读文档。
- 新会话启动前会读取必读文档。
- 可以选择 skill。
- 可以选择模型。
- 可以看到活跃 Codex 进程。
- 可以点击进入线程或会话 CLI。
- 可以从界面输入需求并启动 Codex。
- 整体界面简单，不像完整 IDE。

## 8. 延后功能

以下功能不进入第一版：

- 插件市场。
- MCP 下载中心。
- 内置代码编辑器。
- 云同步。
- 账号系统。
- 多人协作。
- 复杂会话分析。
- 完整 IDE 文件管理。
